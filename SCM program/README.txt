此次数据按照十六进制发送temp数组，数据包temp的定义如下：
//	 	 0xCC   't'    						 t_value    'h'     h_value 	  CRC1 		   CRC2         0xAA

uchar temp[8] = {0xCC,  0x74,   					   00,      0x68,     00,     	   00,   	    00,         0xAA};

//		 包头   ‘t’(表示接下一字节是温度，‘h’同理，是湿度）  温度值     'h'     湿度值    CRC校验码前8位  CRC校验码后8位    包尾

CRC校验是对temp数组前5个字节进行CRC校验，调用函数CRC16(uchar *pData, uchar nData)即可，生成short型16位的校验码，按8位拆分，分别放入到CRC1和CRC2中。
PC串口接收到temp后，对数组进行解包，先检查包头包尾，若包头包尾正确，再对temp的前5个字节进行CRC校验，将生成的校验码和temp数组的CRC1和CRC2对比，如果正确，则无误码，否则丢弃。



上电后，打开PC端的串口助手（V1版本，用串口助手看发送指令，确保指令发送过去后，单片机能正常处理，若能，则说明单片机接收指令解析无误。
			    后面再用PC端写的串口上位机发送，如果可以，则说明串口上位机发送指令无误，若不行，则说明上位机发送指令有问题）：

发送温度湿度：
按十六进制发送：CC 01 01 AA
单片机将发送temp数组到PC，温湿度值非0。

仅发送温度：
按十六进制发送：CC 01 02 AA
单片机将发送temp数组到PC，温度值非0，湿度值为0。

仅发送湿度：
按十六进制发送：CC 02 01 AA
单片机将发送temp数组到PC，温度值为0，湿度值非0。

停止发送：
按十六进制发送：CC 02 02 AA
单片机不再发送数据。

发送其它指令：
单片机不再发送数据。
